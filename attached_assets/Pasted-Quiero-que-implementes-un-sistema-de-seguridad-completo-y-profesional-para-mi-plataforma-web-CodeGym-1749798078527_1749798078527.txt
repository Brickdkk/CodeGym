Quiero que implementes un sistema de seguridad completo y profesional para mi plataforma web CodeGym, que actualmente incluye usuarios registrados, funciones premium, pagos con MercadoPago, y ejecución de ejercicios.

Ya está en funcionamiento, pero necesito que mejores todo el sistema de seguridad, siguiendo los puntos a continuación. No alteres funcionalidades existentes, solo agrega seguridad y haz los test exhaustivos después de cada mejora.

🔒 1. Conexión Segura (HTTPS / SSL)
Asegúrate de que todas las conexiones usen HTTPS con certificado SSL válido.

Si el entorno de Replit no fuerza HTTPS por defecto, implementa un redirect automático de HTTP → HTTPS.

Valida que los formularios de login y pago estén protegidos por HTTPS.

👥 2. Autenticación y Autorización
Refuerza la autenticación de usuarios:

Usa hashing seguro con bcrypt para contraseñas.

Implementa rate limiting en login (para prevenir ataques de fuerza bruta).

Añade verificación de sesión o JWT:

Tokens deben expirar en un tiempo razonable.

Usar firma segura (HS256 o RS256).

Verifica que las rutas premium solo puedan ser accedidas si el usuario tiene suscripción activa o clave válida.

📦 3. Validación y Sanitización
Valida todos los inputs del usuario en frontend y backend (correo, claves, campos de ejercicios, etc).

Aplica sanitización contra XSS e inyecciones (SQL Injection, Command Injection).

Usa una librería como express-validator o similar para backend en Node.js.

🧱 4. Protección contra ataques comunes
Agrega protección contra:

CSRF: usando tokens CSRF en formularios.

XSS: sanitizando inputs y usando headers de seguridad.

Clickjacking: usando X-Frame-Options: DENY

CORS: configura CORS para solo aceptar dominios válidos (ej: https://codegym.cl).

💸 5. Seguridad en pagos (MercadoPago)
Verifica que:

La clave privada (Access Token) esté segura y no expuesta al frontend.

Los Webhooks sean verificados con firma o token secreto.

Los precios no se modifiquen en el frontend (solo validados en backend).

La lógica de activación premium no dependa del frontend.

🧪 6. Test de Seguridad y Logs
Realiza test automáticos (o manuales) que verifiquen:

Accesos no autorizados

Manipulación de datos

Cambios de precios en el frontend

Implementa sistema de logs y alertas (simple) para detectar accesos sospechosos.

Revisa el uso de consola de ejercicios: impide ejecución de código malicioso o comandos peligrosos si aplica.

✅ 7. Checklist final
Toda conexión es HTTPS

Ningún endpoint sensible se expone públicamente

Inputs validados

Sesiones seguras

Funciones premium bien protegidas

Seguridad en pagos y webhooks

Tests exhaustivos pasados

Haz todo esto de forma modular, clara, bien documentada y con pruebas de funcionamiento al final. No avances al siguiente punto hasta que lo anterior esté completamente funcional.
