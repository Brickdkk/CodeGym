# Prompt para Implementación del Sistema de Ejercicios

## Contexto del Proyecto

Necesito integrar un sistema completo de ejercicios de programación en mi proyecto de Replit existente. Este sistema incluye 352+ ejercicios en 5 lenguajes (Python, JavaScript, C, C++, C#) con 4 niveles de dificultad, APIs REST completas y funcionalidad de búsqueda avanzada.

## Archivos del Sistema a Integrar

He preparado 5 archivos principales que contienen todo el sistema:

1. **ejercicios-schema.js** - Contiene esquemas de validación y configuración de niveles
2. **ejercicios-storage.js** - Sistema de almacenamiento en memoria con interfaz completa
3. **ejercicios-autoloader.js** - Carga automática de ejercicios desde repositorios de GitHub
4. **ejercicios-routes.js** - Rutas de API REST con todas las funcionalidades
5. **integracion-servidor.js** - Archivo principal de integración con mi servidor Express

## Archivos de Documentación y Configuración

- **package-additions.json** - Dependencias necesarias (solo 'marked')
- **INSTRUCCIONES-INTEGRACION.md** - Guía detallada paso a paso
- **ejemplo-servidor-completo.js** - Ejemplo funcional de implementación
- **README.md** - Documentación completa del sistema

## Tareas de Implementación Requeridas

### 1. Preparación del Entorno
- Instalar la dependencia 'marked' usando `npm install marked`
- Verificar que Express esté disponible en el proyecto
- Confirmar que el proyecto tiene estructura de servidor Node.js

### 2. Integración de Archivos
- Copiar los 5 archivos .js principales al directorio raíz del proyecto
- Verificar que las importaciones funcionen correctamente
- Confirmar que no hay conflictos con archivos existentes

### 3. Modificación del Servidor Principal
Mi servidor actual está en [ESPECIFICAR ARCHIVO: server.js/app.js/index.js]. Necesito:

```javascript
// Añadir al inicio del archivo (importaciones)
import { integrarSistemaEjercicios } from './integracion-servidor.js';

// Añadir después de configurar middleware pero antes de app.listen()

```

### 4. Verificación de Funcionalidad
Después de la integración, verificar que estas rutas funcionen:

- `GET /api/stats` - Debe devolver estadísticas del sistema
- `GET /api/exercises` - Lista de ejercicios (inicialmente vacía)
- `GET /api/ejercicio/python/hello-world` - Ejercicio específico
- `GET /api/search/hello` - Búsqueda de ejercicios

### 5. Carga Automática de Ejercicios
El sistema debe:
- Cargar ejercicios automáticamente al iniciar (toma 2-3 minutos)
- Mostrar progreso en la consola del servidor
- Cachear ejercicios para evitar descargas duplicadas
- Manejar errores de red graciosamente

## APIs Resultantes Esperadas

### Ejercicios Individuales
- `GET /api/ejercicio/:lenguaje/:nombre` - Obtener ejercicio específico
- Ejemplo: `/api/ejercicio/python/hello-world`

### Listados y Filtros
- `GET /api/exercises` - Todos los ejercicios
- `GET /api/exercises?lenguaje=python` - Por lenguaje
- `GET /api/exercises?nivel=principiante` - Por nivel de dificultad
- `GET /api/exercises/random?cantidad=5` - Ejercicios aleatorios

### Búsqueda
- `GET /api/search/:termino` - Buscar ejercicios por término
- `GET /api/exercises/nivel/:nivel` - Ejercicios por nivel específico

### Estadísticas y Gestión
- `GET /api/stats` - Estadísticas generales del sistema
- `GET /api/logs` - Logs de peticiones recientes
- `GET /api/repositories` - Repositorios configurados
- `POST /api/exercises/load-massive` - Forzar carga masiva
- `POST /api/exercises/refresh` - Refrescar ejercicios existentes

## Especificaciones Técnicas

### Almacenamiento
- Sistema completamente en memoria (no requiere base de datos)
- Cache inteligente para optimizar rendimiento
- Persistencia durante la sesión del servidor

### Rendimiento
- Carga inicial: 2-3 minutos para 352 ejercicios
- Respuesta de APIs: < 100ms para ejercicios en cache
- Manejo de errores: reintentos automáticos para fallos de red

### Formato de Datos
Los ejercicios incluyen:
- `nombre` - Identificador único
- `lenguaje` - python, javascript, c, cpp, csharp
- `titulo` - Título formateado con nivel
- `contenido_md` - Markdown original
- `contenido_html` - HTML renderizado
- `secciones` - Objeto JSON con secciones parseadas
- `github_url` - URL del archivo fuente

## Manejo de Errores Esperado

### Errores Comunes y Soluciones
1. **Module not found 'marked'** → `npm install marked`
2. **Las rutas no responden** → Verificar await antes de integrarSistemaEjercicios
3. **Ejercicios no cargan** → Esperar 2-3 minutos, verificar logs de consola
4. **Conflictos de rutas** → El sistema usa prefijo /api/ para evitar conflictos

### Logs de Progreso
